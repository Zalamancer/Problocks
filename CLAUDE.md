## What This Project Is

AI-powered game creation platform. Students describe a game → Claude orchestrates AI tools (PixelLab, Meshy, Suno, ElevenLabs) → playable HTML5 game → published to marketplace → students earn from plays.

**Not a game engine.** No Three.js, no physics, no custom scripting runtime. Games are HTML5 output generated by AI.

## Stack

- **Next.js 16** (App Router) — studio + marketplace + classroom in one app
- **React Flow** (@xyflow/react) — node-based studio canvas
- **Zustand** — client state
- **Supabase** — auth + database + storage
- **Tailwind CSS v4** — styling
- **Railway** — hosting

**All UI is built from these components. Do not invent new buttons, inputs, selects, or toggles.**

Ported directly from ProAnimate. Import from `@/components/ui`:

```
src/components/ui/
  panel-controls/         ← primary building blocks for all panels
    PanelSection          — collapsible section with title + border
    PanelSlider           — drag-to-scrub numeric input (click to type, wheel to adjust)
    PanelSelect           — portal dropdown with keyboard nav
    PanelInput            — text input with optional label + icon
    PanelTextarea         — resizable textarea with optional char count
    PanelToggle           — switch toggle with label
    PanelCheckbox         — checkbox with label
    PanelButtonGroup      — segmented button group (icon or label)
    PanelActionButton     — primary/secondary/destructive/accent button
    PanelSearchInput      — search input with icon
    PanelCategoryTabs     — horizontal scrollable tab row (text pills, bg-accent active)
    PanelIconTabs         — animated expanding icon+label pill bar (bg-white active, flex trick) ← use for top-of-panel tab bars like All/Images/Audio
    PanelDropZone         — drag-and-drop upload area
    PanelMultiSelect      — multi-select dropdown
    tokens.ts             — PANEL design token constants
  IconButton              — icon-only button (solid/ghost/outline, sm/md/lg)
  TabNavigation           — tab bar (animated or standard)
  GlassPanel              — frosted glass container
  LoadingSpinner          — spinner + LoadingOverlay
  ConfirmDialog           — global confirm modal (mount in layout)
  ToastContainer          — global toast system (mount in layout)
```

**Stores for global UI:**
- `@/store/toast-store` — `useToastStore` → `addToast(type, message)`
- `@/store/confirm-dialog-store` — `useConfirmDialogStore` → `confirm({ title, description })`

**When porting components from AutoAnimation (`/Users/ihsanduru/autoStudio/AutoAnimation/src/`):**
Read the full source file — not just the className colors. The animated flex-expand pill tab bar (`PanelIconTabs`), pointer-lock slider, portal dropdown, etc. all have non-obvious behavioral patterns that won't be visible from a style scan. Copying styles alone without the underlying structure is wrong.

**Design tokens** (defined in `globals.css`, used via Tailwind):
- `bg-panel-surface` / `bg-panel-surface-hover` / `bg-panel-bg`
- `border-panel-border`
- `bg-accent` / `text-accent` / `hover:bg-accent-hover`
- `z-dropdown` / `z-modal` / `z-toast`

## Code File Organization

**Max 300–500 lines per file — applies to all files including `.md` docs.** Split at component/section boundaries for code, and at topic boundaries for docs, never mid-thought.

**One component = one file.** Never put multiple exported components in the same file.

**Group by what they do** (not just by app):

```
src/components/studio/
  panels/         ← individual left-panel tab contents
  layout/         ← shell components (StudioLayout, TopMenuBar)
  canvas/         ← canvas and node components
  modals/         ← dialog/modal components (when added)

src/hooks/        ← custom React hooks extracted from components
src/store/        ← one Zustand store per domain (studio, auth, project)
src/lib/          ← pure utilities, no React
```

**AI context rule:** When Claude edits a file, only send that file + any direct imports it needs — never the full codebase. Use tool calls to read specific files on demand.
